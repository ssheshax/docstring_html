libs.core.reboot_helper.get_test_suite(class_name: Optional) → Optional

Returns a customized test suite based on the context of a reboot scenario and the structure of the test class.

This method determines how the test cases are added to the test suite depending on whether:

  1. The current scenario is a reboot (as determined by is_reboot_scenario()).

  2. The number of test cases defined in the class.

If a reboot scenario is detected:

  • It identifies the function to be treated as the “callee” (the function that initiates the reboot).

  • The method then determines the line number where the callee function starts.

  • If there are multiple test cases defined in the class, it constructs a custom test suite that includes: - The callee test case. - Any subsequent test cases that are defined after the callee in the class.

  • If only one test case is defined, the test suite is created only for the callee test case.

If no reboot scenario is detected:

  • If there are multiple test cases, they are ordered by their starting line numbers and added to the suite.

  • If only one test case is defined, the default test suite for the class is returned.

Parameters:

class_name (Optional[type]) – The name of the test class. This should be a class type that is used for test cases.

Returns:

A test suite with the appropriate test cases ordered as required by the reboot scenario or default order.

Return type:

unittest.TestSuite or None

libs.core.reboot_helper.is_reboot_scenario() → bool

Determines whether the current scenario is a valid reboot scenario.

This function checks the reboot context, which is retrieved through the __get_reboot_context() function. If a valid reboot context is found (i.e., is_valid_reboot is True), it returns True, indicating that the current scenario is a reboot scenario.

If the reboot context is invalid (either the context is None or is_valid_reboot is False), the context is cleaned up by calling the __remove_reboot_context() function, and the function returns False.

Returns:

True if the scenario is a valid reboot scenario, otherwise False.

Return type:

bool

libs.core.reboot_helper.reboot(self, callee: str, data=None) → bool

Initiates a system reboot by setting the reboot context and invoking the system shutdown command.

This method manages the reboot process by storing relevant information in the reboot context and attempting to trigger a system reboot. It also handles storing and validating the reboot context, and managing the test execution if necessary.

Parameters:

  • callee (str or None) – The name of the callee function or entity requesting the reboot. This is stored in the reboot context. If None, an error is logged.

  • data (any, optional) – Any additional data to store in the reboot context. Default is None.

Returns:

True if the reboot process was initiated successfully; False if there was an error in setting up the reboot context or in initiating the reboot.

Return type:

bool

Notes:

  • This function first ensures that a reboot context exists or creates one if necessary.

  • It logs the caller’s function name from the stack trace.

  • The callee is checked for validity, and if None, the function logs an error and returns False.

  • The reboot context is stored using the __store_reboot_context method. If storing fails, it logs an error and returns False.

  • If the current test result is not None, the function will stop the test before initiating the reboot.

  • The system reboot command is triggered via the shutdown.exe command.

  • The code pauses waiting for user input (which should be replaced with an actual reboot trigger).

  • The function is designed to terminate after initiating the reboot, so code following the reboot should not be executed.

libs.core.reboot_helper.setup(func)

A decorator that sets up the test environment for a test method in a reboot scenario.

This decorator wraps the test setup function and ensures that a reboot context is managed correctly during test execution. It performs the following actions:

  • Initializes a reboot context before running the test.

  • Restores a previously saved reboot context if a reboot scenario is detected.

  • Marks the reboot as invalid if an unexpected reboot is encountered.

  • Attempts to restore the unittest state (errors, failures, and class data) after a reboot.

  • Executes the original func (test method) and handles any exceptions that may occur during setup.

  • If a test method decorated with @test is present, it triggers a system reboot and continues the test execution.

Parameters:

func (Callable) – The test method to be wrapped by the decorator.

Returns:

A wrapped version of the func that includes the setup logic for reboot scenarios.

Return type:

Callable

libs.core.reboot_helper.teardown(func)

A decorator to perform teardown actions after a test method completes.

This decorator wraps a test method and provides a mechanism to execute specific actions after the test runs. In this case, it checks if the current test method is the last one in a given list of test cases. If so, it removes a reboot context by calling __remove_reboot_context(), logs the removal, and then executes the original test method. If the current test method is not the last in the list, the decorated function will not be called.

This decorator is typically used for post-test cleanup operations, such as resetting or removing context or performing any other necessary actions after a test completes.

Parameters:

func (Callable) – The test function to be wrapped by the decorator.

Returns:

A wrapped version of func that will execute the necessary teardown actions before invoking the original function.

Return type:

Callable
